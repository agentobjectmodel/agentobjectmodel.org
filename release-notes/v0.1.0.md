# Agent Object Model™ v0.1.0

**Status:** Current, unsigned payloads only.  
**Spec:** https://www.agentobjectmodel.org/spec/v0.1.0/  
**Related:** [CHANGELOG.md](../CHANGELOG.md) · [RELEASING.md](../RELEASING.md)

## Core spec
- Defined the Agent Object Model™ v0.1.0: surfaces, actions, navigation, and automation policies (`allowed`, `open`, `forbidden`).
- **Required fields**: Every surface must include `automation_policy`, `aom_version`, `surface_id`, `surface_kind`, `generated_at` (ISO 8601), plus `purpose`, `context`, `tasks`, `entities`, `actions`, `state`, `navigation`, `signals`. Optional: `generator`, `calling_agent` (agent identity / traceability), `a2h`.
- Output schema supports optional `agent_id`, `agent_name`, `key_issuer` for traceability; the same values are sent in the action-invocation request to the site when the surface requires them.
- Documented the well‑known site policy at `/.well-known/aom-policy.json`.
- Signed/secure payloads and verification APIs are **out of scope** for this repo.

## Examples
- Added `examples/v0.1.0/` with runnable flows (e.g. `login-single`, `ecom-flow`).
- Minimal valid surface (forbidden page template) at `examples/v0.1.0/_forbidden-page-template/aom-policy.forbidden.page.aom.json`; copy-paste for page-level no-automation.
- Standardized `automation_policy` and navigation patterns for v0.1.0.
- Provided golden `*.output.json` files plus `.json.skip` markers where outputs are hand‑edited.

## Validators and tooling
- Python and Node validators for **inputs** (`*.aom.json`), **outputs** (`*.output.json`), and **site policies**.
- Bulk validators (`validate_all`) that walk `examples/v0.1.0/`, validate both inputs and outputs (including `.json.skip`), and report JSON parse errors and schema violations with paths and JSON snippets.
- `create-outputs` tools for generating canonical `*.output.json` files from demo agents, respecting `.json.skip` markers.

## CLI entrypoints
- **Python:** `aom.py` · **Node:** `aom.mjs`
- Commands: `validate input|output|site`, `validate all`, `create-outputs`, `demo run` / `demo test`

## Demo agents
- Python and Node demo agents under `examples/v0.1.0/demo-agents/`.
- Support `--folder v0.1.0/<example>` and `--test-case <stem>`; full test suites (pytest, Node `node --test`).

## Plugins and integrations
- Reference plugins (WordPress, Next.js, Nuxt 3, Gatsby, static HTML, Shopify Liquid) and the AOM Surface Explorer browser extension are distributed from [aom.tools](https://aom.tools). They target v0.1.0; see the Community and Support section of the root README for direct download links.
